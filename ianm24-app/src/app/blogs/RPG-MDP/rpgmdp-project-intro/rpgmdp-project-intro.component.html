Getting a machine to optimally play through a turn-based RPG: RPG-MDP Project Intro

Posted 20 May 2020 5:00PM EST

The idea behind the Role-Playing Game as a Markov Decision Process (RPG-MDP) was proposed by my friend Julian. He thought that if we could represent a turn-based RPG as a Markov Chain, then we could use machine learning to get the computer to find the most optimal path throughout the RPG. More specifically, and possibly ambitiously, we wanted to see if we could get the computer to play through a Pokemon game. Another outcome of this project is to be able to tell whether certain battles are balanced and if a certain set of paths are balanced as far as speed is concerned

While that is our long-term hope, the final product may end up looking a bit different. If you want, you can look at the code in the GitHub repository here. The project is coded in Python 3.8. I hope you will enjoy the development logs to come!
